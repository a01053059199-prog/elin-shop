# ELIN 도메인 적용 및 배포 안내

현재 쇼핑몰은 로컬 주소 `http://localhost:4173`에서 실행됩니다.
실제 도메인으로 접속하려면 아래 순서가 필요합니다.

## 1. 도메인 준비

도메인을 구매합니다.

예시:

- `elin.co.kr`
- `elinshop.kr`
- `elin-store.com`

도메인 구매 후 DNS 관리 화면에 들어갈 수 있어야 합니다.

## 2. 서버 배포

이 프로젝트는 Node.js 서버입니다.
아래 중 하나에 올릴 수 있습니다.

- VPS 서버
- Cafe24 Node.js 호스팅
- Render
- Railway
- AWS Lightsail
- Naver Cloud

배포 서버에서 실행 명령은 다음과 같습니다.

```bash
npm start
```

서버 환경변수는 실제 운영용으로 바꿔야 합니다.

```bash
PORT=4173
ADMIN_USER=admin
ADMIN_PASSWORD=강한비밀번호
ADMIN_PIN=강한관리자핀
```

## 3. DNS 연결

서버에서 받은 주소에 따라 DNS를 설정합니다.

### 서버 IP가 있는 경우

도메인 DNS에 아래처럼 입력합니다.

| 타입 | 이름 | 값 |
| --- | --- | --- |
| A | @ | 서버 IP 주소 |
| A | www | 서버 IP 주소 |

예시:

```text
A    @      123.123.123.123
A    www    123.123.123.123
```

### 배포 서비스 주소가 있는 경우

Render/Railway 같은 서비스에서 받은 주소를 연결할 때는 보통 CNAME을 씁니다.

| 타입 | 이름 | 값 |
| --- | --- | --- |
| CNAME | www | 배포 서비스 주소 |

루트 도메인 `@`는 서비스에서 안내하는 A 레코드 또는 ALIAS/ANAME 값을 사용합니다.

## 4. SSL 적용

실제 판매용 쇼핑몰은 반드시 HTTPS가 필요합니다.

배포 서비스에서 Custom Domain을 추가하면 보통 SSL이 자동 발급됩니다.
VPS를 직접 쓰는 경우에는 Nginx와 Let's Encrypt를 사용합니다.

## 5. 접속 주소 변경 확인

DNS 전파 후 아래 주소로 접속합니다.

```text
https://내도메인/
https://내도메인/admin.html
```

## 6. 실제 운영 전 체크리스트

- 관리자 비밀번호 변경
- 사업자 정보 입력
- 통신판매업 신고 정보 입력
- 개인정보처리방침 추가
- 이용약관 추가
- 반품/교환 정책 추가
- PG 결제 연동
- 주문 데이터 백업 구조 적용

## 내가 실제로 연결하려면 필요한 정보

아래 정보가 있어야 도메인 연결 값을 정확히 정할 수 있습니다.

- 구매한 도메인 이름
- 도메인 구매처 또는 DNS 관리처
- 배포할 서버/호스팅 서비스
- 서버 IP 또는 배포 서비스에서 제공한 target 주소
